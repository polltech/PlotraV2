Authentication

All API requests require authentication using Bearer tokens obtained through the login process.

Getting Started

  1. Login via the web interface to obtain an access token
  2. Include the token in all API requests using the Authorization header
  3. Tokens expire after 30 minutes of inactivity
POST /api/v2/auth/token-form

Authenticate user and obtain access token

Parameters
Parameter Type Required Description
username string Yes User email address
password string Yes User password
Example Request
curl -X POST http://localhost:8000/api/v2/auth/token-form \
-d "username=user@example.com&password=password123"
Success Response (200)
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "expires_in": 1800
}
GET /api/v2/auth/me

Get current authenticated user information

Headers
Authorization: Bearer YOUR_ACCESS_TOKEN
Success Response (200)
{
  "id": "user-uuid",
  "email": "user@example.com",
  "first_name": "John",
  "last_name": "Doe",
  "role": "farmer",
  "country": "Kenya"
}

Farms API

Manage farm information and operations.

GET /api/v2/farmer/farm

Get user's farm information

Returns farm details for the authenticated farmer user.

POST /api/v2/farmer/farm

Create a new farm

Request Body
{
  "name": "Mountain View Farm",
  "location": "Kiambu County",
  "size_hectares": 25.5,
  "description": "High altitude arabica coffee farm"
}

Parcels API

Manage individual land parcels within farms.

GET /api/v2/farmer/farm/{farm_id}/parcels

Get all parcels for a specific farm

POST /api/v2/farmer/farm/{farm_id}/parcels

Create a new parcel within a farm

Request Body
{
  "code": "P001",
  "area_hectares": 2.5,
  "land_use_type": "agricultural",
  "coordinates": [[x1,y1], [x2,y2], ...]
}

EUDR Compliance API

Generate and manage EU Deforestation Regulation compliance documents.

POST /api/v2/admin/eudr/dds

Generate a Due Diligence Statement

Request Body
{
  "farm_ids": ["farm-uuid-1", "farm-uuid-2"],
  "commodity_type": "Coffee",
  "quantity": 1000,
  "country_of_origin": "Kenya"
}
GET /api/v2/admin/eudr/dds/{dds_id}

Retrieve a specific DDS document

Users API

Manage user accounts and permissions (Admin only).

GET /api/v2/admin/users

List all users in the system

GET /api/v2/admin/cooperatives

List all cooperatives

Error Codes

Common API error responses and their meanings.

Status Code Description Common Causes
400 Bad Request Invalid parameters, missing required fields
401 Unauthorized Missing or invalid authentication token
403 Forbidden Insufficient permissions for the operation
404 Not Found Resource doesn't exist or wrong endpoint
500 Internal Server Error Server-side issue, contact support

Error Response Format

{
  "detail": "Error message description",
  "type": "error_type" // optional
}
